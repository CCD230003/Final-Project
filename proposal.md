## Description
My project will pose as inspiration for others in the same field I am in that wish to create and share a story through code. I would also like for it to be an inspiration; 
proof that if I can do it, then anyone else can do it too.

## Features
- Feature 1
	- Multiple sprites of multiple different characters; assuming it'll be implemented in the same type of way the popup tab is made with pygame, layering things
 - over each other.
- Feature 2
	- A few different backgrounds that swap out with each other when needed; I think I can get it to work again like the pygame black screen surface object.
- Feature 3 
	- A dialogue box with dialogue that moves as though its being typed; I'm not sure how this one will work, but I feel like with a couple of classes or
 - main functions with the time keyword, I could get it to be animated on the screen.

## Challenges
- I need to look into how to get mouse input to work in my code and be considered while the player is playing the visual novel.
- Research how other people have made their visual novels and if I need to install new libraries I don't have already.
- Need to look into how to swap between scenes or sprites with either timing or mouse input.

## Outcomes
Ideal Outcome:
- A relatively short (~2-3 minute) visual novel with three sprites that talk back and forth with at least two backgrounds to signify movement happening in the story.

Minimal Viable Outcome:
- At minimum, a short (~1-2) visual novel with interation between two characters on one background would be acceptable.
